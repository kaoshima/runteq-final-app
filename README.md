## サービス概要

QAエンジニア（テスト担当者）向けの、**テスト設計・準備の効率化ツールキット**です。
文字入力の検証で利用可能なテキスト生成、柔軟な連番データ作成、境界値・同値クラスの洗い出しなど、テストに必要な面倒な準備作業を自動化・サポートします。
面倒な「考える作業」や「ツールを探す手間」をなくし、本来のテスト業務に集中できる環境を提供します。

## このサービスへの思い・作りたい理由

私自身がQAエンジニアとして働く中で、日々のテスト業務には多くの「準備作業」が伴います。
「500文字上限」のテストのために文字数カウントツールを探し、「全角ひらがな」のテキストを用意する。「ダミーのユーザーID」を`test001`, `test002`...と手動で考える。「年齢制限の境界値」を頭で計算し、仕様書から「同値クラス」を洗い出す...。
これらはテストそのものではありませんが、確実に時間と認知コストを奪っていきます。特に、複数のツールやExcelを何度も行き来する作業は、**「面倒」であり、「時間がもったいない」** と強く感じていました。

**「なぜ、この単純作業を毎回手動でやっているんだろう？」**
**「これらのツールが、一つの画面にまとまっていたら最高なのに。」**

この現職での切実な課題を、学んでいるプログラミング技術で解決したい。そして、自分と同じように感じているQA担当者や、テスト作業に手間を感じている開発者の助け（アシスト）になりたいと思い、このサービスを開発することにしました。

## ユーザー層について

* **メインターゲット:** Webサービスやアプリ開発の現場にいる**QAエンジニア**。
    * **理由:** 私自身がそうであるように、QAエンジニアは日々、仕様書と格闘しながら、多種多様なテストデータを手動で考える非効率さに直面しています。そのため、このツールが解決する課題に最も強く共感し、価値を感じてもらえる層だと考えました。
* **サブターゲット:** **個人開発者**や、QA専任者がいない小規模チームの**Webデベロッパー**。
    * **理由:** 開発者が自分でテストを行う際も、テストデータの準備は同様に発生します。その工数を削減する「かゆいところに手が届く」ツールとして利用してもらえると考えました。

## サービスの利用イメージ

あるQAエンジニアが、新しく実装された「会員登録フォーム」のテストを行うシーンを想定します。

1.  エンジニアは、仕様書で「氏名：50文字以内, 全角ひらがなのみ」「ユーザーID：20文字以内, 半角英数」「年齢：20歳以上」という項目を確認します。
2.  **このサービスを利用し、**
    1.  「バリデーションテキスト生成」機能で、「50文字」「全角ひらがなのみ」を指定。有効なひらがな50文字だけでなく、無効なカタカナ 半角英数 51文字や、エッジケースの空文字なども即座にコピーします。
    2.  「**柔軟なデータ生成**」機能で、「user- (連番3桁)」と指定。「user-001」「user-002」...のリストを生成し、コピーします。
    3.  「**テストクラス分析**」機能で、「20」（下限のみ）と入力。境界値「19, 20, 21」と同値クラス「19以下(無効)」「20以上(有効)」を即座に確認します。
3.  これらのコピーした値を、テスト対象のフォームに素早く貼り付け、テストを実行します。

これにより、ツールを探したり、頭で考えたり、手動でコピペを繰り返したりする**「思考の中断」がなくなり**、テスト実行のテンポを崩さずに業務を進められます。

## サービスの差別化ポイント・推しポイント

* **競合:** 文字数カウントツール、ダミーデータ生成サイト、OGP確認サイト、Excel（手動でのテストケース作成）など、**機能が分散した単体ツール**。
* **差別化ポイント（推しポイント）:**
    QA業務で頻繁に必要となる**複数の機能（データ生成、テスト設計支援）をワンストップで提供**することです。
* **優れている点:**
    多くの単体ツールやExcel作業では、テスト担当者が都度ツールを探したり、仕様書から手動でパターンを洗い出す必要がありました。
    本サービスは「QAエンジニアの道具箱」として、**単純なデータ生成**から、**同値分割や境界値分析といった「テスト設計」の領域**までを一箇所でサポートします。これにより、**コンテキストスイッチ（思考の切り替え）のストレスと時間を徹底的に削減**します。

## 機能候補

### MVPリリース時に作っていたいもの（核となる機能）

* **① バリデーションテキスト生成機能**
    * **文字数テスト:** 指定した文字数（例: `50`）に対し、「`50`文字」「`51`文字（上限超え）」「`49`文字（上限-1）」のデータを生成。
    * **文字種テスト:** 仕様（例: `半角英数のみ`）に対し、**有効なデータ**（`abc123`）と**無効なデータ**（`あああ`, `！＠＃`）を自動で生成。
    * **エッジケース:** 「空文字（""）」「半角スペースのみ」「全角スペースのみ」のデータをワンクリックで生成・コピーできる。
* **② 柔軟なデータ生成機能**
    * 指定したプレフィックス（接頭辞）やフォーマットに基づき、連番データを生成する。（例: `user-` と入力 → `user-001`, `user-002`, `user-003`...）
* **③ シンプルなダミーデータ生成機能**
    * 「日本人の姓名」「ランダムなメールアドレス」「ランダムな電話番号」など、よく使う情報をワンクリックで生成・コピーできる。
* **④ テストクラス分析サポート機能**
    * 仕様値（例: 「10〜100」や「20以上」）を入力すると、テストすべき**境界値**（例: `9, 10, 11`, `99, 100, 101`）と**同値クラス**（有効/無効）をリストアップする。

### 本リリースまでに作っていたいもの（追加機能）

* **⑤ OGP確認機能**
    * テスト対象のURLを入力すると、そのページの `og:title` `og:image` などのメタ情報を取得・表示し、プレビューできる機能。
* **⑥ 組み合わせパターン表（ペアワイズ法）生成機能**
    * 因子と水準（例: OS: Win/Mac, ブラウザ: Chrome/Safari）を入力すると、テストすべき最小限の組み合わせパターン（ペアワイズ法）を表として自動生成する。
* **⑦ デシジョンテーブル作成支援機能**
    * 「条件」と「動作」をフォームに入力すると、デシジョンテーブル（条件網羅のマトリクス表）を自動生成する。
* **⑧ セキュリティテスト用文字列生成機能**
    * XSS（クロスサイトスクリプティング）やSQLインジェクションの基本的なテストパターン文字列を生成する。
* **（将来的）ユーザー登録機能 (Devise)**
    * 作成したデシジョンテーブルやパターン表、よく使うダミーデータセットをプロジェクトごとに保存・管理できるようにする。

## 使用する技術スタック

* **使用するフレームワーク:** Ruby on Rails (バージョン 7.x系を想定)
* **データベース:** PostgreSQL (Herokuでのデプロイを想定)
* **デプロイ先:** Heroku
* **使用予定のライブラリ:**
    * `Devise` (本リリースでユーザー機能を実装する場合)
    * `Nokogiri` (本リリースのOGP確認機能で、対象URLのHTMLを解析するために使用)
* **一般的なCRUD以外の実装予定の機能について:**
    * 本サービスの中心は、データベースへの保存（CRUD）を伴わない「データ生成」と「ロジック処理」です。
    * フォームからパラメータ（文字数や範囲など）を受け取り、Railsのコントローラ、あるいはロジックを分離した**サービスクラス（Service Object）**内でRubyのロジックを実行し、結果をビューに表示するという実装が中心になります。
    * **技術的チャレンジ（本リリース）:**
        * **OGP確認機能**: 外部サイトへのHTTPリクエスト（`Net::HTTP`や`Faraday`などを使用）と、取得したHTMLの`Nokogiri`による解析。
        * **デシジョンテーブル・組み合わせ機能**: 複雑なテスト技法のアルゴリズムをRubyで正確に実装すること。
        * **セキュリティテスト用文字列**: XSSやSQLインジェクションのパターンを安全に生成・表示するロジックの実装。
